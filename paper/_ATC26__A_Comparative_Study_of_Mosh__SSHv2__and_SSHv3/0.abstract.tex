\begin{abstract}
Remote shells are increasingly used by automation workflows that repeatedly issue commands, parse terminal output, and react to visible feedback. A common expectation is that newer remote-control protocols should improve this loop: SSH3 inherits QUIC's fast connection setup and stream multiplexing, while Mosh avoids byte-stream stalls by synchronizing terminal state over UDP. However, these design advantages do not imply that one protocol is uniformly better for all terminal interactions. This paper presents a measurement study of SSHv2, Mosh, and SSH3 under deterministic AI-like interactive terminal workloads. The benchmark does not run a full AI agent; instead, it isolates the transport-facing feedback channel using small-output command loops, large-output fixtures, and keystroke-to-echo probes across a VPN path and three controlled LAN impairment profiles. Our results show that the best protocol depends strongly on the workload. SSH3 provides the fastest session setup and the lowest small-output command latency in all scenarios. Mosh provides the lowest visible keystroke latency, especially under impairment, but receives only 1.24--4.19\% of the expected bytes for large-output fixtures because it synchronizes screen state rather than complete byte streams. SSHv2 and SSH3 preserve byte-complete and content-correct output for successful large-output samples, but SSHv2 is faster than SSH3 in the comparable byte-complete W2 cases. These results show that remote-shell protocol selection for AI-like interactive clients must account for the dominant feedback pattern: setup, short command loops, visible editing responsiveness, or complete output delivery.

\end{abstract}
